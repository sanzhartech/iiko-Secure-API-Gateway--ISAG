# ISAG — Руководство по конфигурации окружения

Шлюз безопасности **iiko Secure API Gateway** загружает и проверяет все настройки при запуске из переменных окружения или локального файла `.env` с использованием механизма Pydantic `BaseSettings`.

Если какая-либо из обязательных переменных отсутствует или имеет неверный тип, приложение отказывается запускаться, обеспечивая **безопасное при отказе (fail-secure)** развертывание.

---

## 1. Справочник переменных окружения

Ниже приведен полный список переменных окружения, определенных в схеме конфигурации:

| Имя переменной | Тип данных | Значение по умолчанию | Описание |
| :--- | :--- | :--- | :--- |
| **`APP_ENV`** | `development` \| `production` | `development` | При значении `production` принудительно используется HTTPS для вышестоящего API (upstream), отключается интерактивная документация Swagger UI и требуется явная настройка CORS. |
| **`APP_DEBUG`** | `boolean` | `false` | Включает отладочный уровень логов и вывод трассировки ошибок (должно быть `false` в продакшене). |
| **`APP_HOST`** | `string` | `0.0.0.0` | Сетевой IP-адрес интерфейса для привязки сервера FastAPI. |
| **`APP_PORT`** | `integer` | `8000` | Порт, на котором шлюз ожидает входящие запросы. |
| **`DATABASE_URL`** | `string` | `sqlite+aiosqlite:///./gateway.db` | URL-адрес подключения к базе данных SQLAlchemy. В продакшене используйте `postgresql+asyncpg://...`. |
| **`REDIS_URL`** | `string` | `""` | URL-адрес подключения к Redis (например, `redis://redis:6379/0`). Требуется для ограничения частоты запросов и защиты от повторных атак по JTI. |
| **`JWT_PRIVATE_KEY_PATH`** | `string` (Путь) | `./keys/private.pem` | Путь к закрытому ключу RSA, используемому для подписи токенов шлюзом. |
| **`JWT_PUBLIC_KEY_PATH`** | `string` (Путь) | `./keys/public.pem` | Путь к открытому ключу RSA по умолчанию, используемому для проверки токенов. |
| **`JWT_PUBLIC_KEYS_PATH`** | `string` (Путь) | `./keys/public_keys.json` | Путь к реестру открытых ключей, связывающему Key ID (KID) с их PEM-содержимым. |
| **`JWT_ACTIVE_KID`** | `string` | `default` | Активный Key ID, записываемый в заголовок выпускаемых токенов. |
| **`JWT_ACCESS_TOKEN_EXPIRE_MINUTES`** | `integer` | `60` | Время жизни access-токенов (рекомендуется: `15`). |
| **`JWT_REFRESH_TOKEN_EXPIRE_DAYS`** | `integer` | `7` | Время жизни refresh-токенов. |
| **`JWT_CLOCK_SKEW_SECONDS`** | `integer` | `60` | Максимальная погрешность для расхождения системных часов (не более `300`). |
| **`IIKO_API_BASE_URL`** | `string` | `http://localhost:8000` | Базовый URL вышестоящего iiko API (в продакшене должен начинаться с `https://`). |
| **`IIKO_API_KEY`** | `string` | **Обязательно** | Секретный API-ключ, используемый шлюзом для авторизации запросов в самом iiko API. |
| **`GATEWAY_CLIENT_SECRET`** | `string` | **Обязательно** | Секретная соль, используемая шлюзом для проверки подлинности учетных данных клиентов при выпуске токенов. |
| **`RATE_LIMIT_PER_IP`** | `string` | `100/minute` | Лимит запросов по умолчанию для неавторизованных IP-адресов. |
| **`RATE_LIMIT_PER_USER`** | `string` | `50/minute` | Лимит запросов по умолчанию для авторизованных клиентов шлюза. |
| **`RATE_LIMIT_AUTH_ENDPOINT`** | `string` | `10/minute` | Лимит частоты запросов для маршрутов получения и обновления токенов. |
| **`CORS_ALLOWED_ORIGINS`** | `string` (CSV) | `http://localhost:3000...` | Список разрешенных CORS-источников, разделенных запятыми. |
| **`CORS_ALLOW_CREDENTIALS`** | `boolean` | `true` | Разрешает браузерным запросам отправлять авторизационные файлы cookie и заголовки. |
| **`TRUSTED_PROXY_CIDRS`** | `string` (CSV) | `""` | Список доверенных обратных прокси-серверов в формате CIDR через запятую (например, `172.16.0.0/12`). |
| **`LOG_LEVEL`** | `DEBUG` \| `INFO` \| `WARNING` \| `ERROR` | `INFO` | Детализация вывода логов. |
| **`LOG_FORMAT`** | `json` \| `console` | `json` | Формат вывода логов (структурированный JSON или консольный вид). |
| **`ADMIN_USERNAME`** | `string` | `None` | Если задано, при запуске создает учетную запись администратора с этим идентификатором. |
| **`ADMIN_PASSWORD`** | `string` | `None` | Пароль для создаваемой при запуске учетной записи администратора. |

## 2. Разделение секретов: Upstream vs Gateway Client Secret

Важнейшим аспектом безопасности является строгое разделение переменных **`IIKO_API_KEY`** и **`GATEWAY_CLIENT_SECRET`**:
*   **`IIKO_API_KEY`**: Это приватный ключ, предоставляемый платформой iiko ERP. Он идентифицирует сам шлюз при выполнении проксируемых вызовов к вышестоящему API.
*   **`GATEWAY_CLIENT_SECRET`**: Это отдельный секретный ключ, используемый шлюзом для проверки подлинности сторонних клиентов, которые запрашивают JWT-токены у шлюза.

**Почему их нельзя объединять или использовать одно и то же значение:**
Использование ключа iiko API в качестве секрета клиента шлюза создает единую точку отказа. Если сторонний клиент скомпрометирует секретный ключ, который он использует для связи со шлюзом, он мгновенно получит прямые административные учетные данные для доступа к iiko API в обход всех лимитов запросов, аудита и контроля доступа ISAG. Валидаторы Pydantic при запуске принудительно проверяют, чтобы эти два значения различались.
